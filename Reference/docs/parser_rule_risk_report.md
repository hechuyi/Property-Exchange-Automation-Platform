# 解析器规则误抓风险报告

> 2026-03-23 说明：本报告的风险条目已并入 `docs/real_operation_test_report_2026-03-23.md` 第 9 节；本文件保留用于追溯原始专项分析上下文。

## 1. 目标与范围
- 目标：评估“解析器可运行但规则可能抓错字段”的风险。
- 范围：`parsers/` 下各交易所解析器与关键兜底逻辑（不评估下载器）。
- 说明：本报告关注“误提取/串字段”，不是“程序崩溃”。

## 2. 结论摘要
- 当前解析链路可正常跑通，但存在多处“规则过宽”的误抓风险。
- 最高优先级风险集中在：
  1. 北京解析器的地区与转让方兜底规则。
  2. 上海增资页面“名称”标签的融资方识别。
  3. 深圳项目编号的宽松兜底模式。
  4. 广州类型与集团推断中的文本启发式。

## 3. 高优先级风险（建议先修）

### 风险 A：北京“所在地区”正则分组歧义，可能抓错或异常
- 位置：`parsers/beijing_standard.py:600`
- 规则：
  - `re.search(r'存放于|位于|目前存放于\s*([^<\n]{2,20}?)(?:市|区|县)', text)`
- 问题：
  - 捕获组只绑定在第三个分支（`目前存放于`）上。
  - 命中“存放于/位于”时 `group(1)` 可能为空，导致地区抓错或出现异常路径。
- 影响字段：`所在地区`
- 风险级别：高
- 建议：
  - 改为统一包裹前缀分支：`(?:存放于|位于|目前存放于)\s*(...)`
  - 抓取后增加 `if match and match.group(1)` 再写入。

### 风险 B：北京把“联系人”回填到“转让方”
- 位置：`parsers/beijing_standard.py:617`, `parsers/beijing_standard.py:620`
- 规则：
  - 当页面包含“与转让方联系/转让方统一组织”时，回退抓取 `联系人: xxx` 并写入 `转让方`。
- 问题：
  - “联系人”语义与“转让方”不等价，属于跨字段污染。
- 影响字段：`转让方`
- 风险级别：高
- 建议：
  - 删除该回退；或仅写入 `经办人`/备注，不写 `转让方`。

### 风险 C：上海增资项目中“名称”标签过宽，易误写融资方
- 位置：`parsers/shanghai_standard.py:158`, `parsers/shanghai_standard.py:267`, `parsers/shanghai_standard.py:272`
- 规则：
  - `_is_company_name_label()` 在 `idx == 0` 时直接视为公司名称标签。
- 问题：
  - 任何首列“名称”行都可能被识别为融资方（包括机构名称、其他表头名称）。
- 影响字段：`融资方`
- 风险级别：高
- 建议：
  - 仅允许受限上下文（如“基本情况/企业基本情况”区块）触发。
  - 增加否定词过滤（交易所、受托机构、联系人等）。

### 风险 D：深圳项目编号兜底模式包含通用 `CQ\d+`
- 位置：`parsers/shenzhen.py:18`, `parsers/shenzhen.py:180`
- 规则：
  - `(?:[GQ][36R]...|CQ\d{8,}(?:-\d+)?)` 用于全页兜底提取项目编号。
- 问题：
  - 页面中非项目编号（工单号、编号文本）也可能匹配 `CQ\d+`。
- 影响字段：`项目编号`
- 风险级别：高
- 建议：
  - `CQ` 模式仅在明确标签上下文（“项目编号/国资监测编号”相邻）启用。
  - 全页兜底优先限定交易所前缀格式，降低自由文本匹配权重。

## 4. 中优先级风险

### 风险 E：广州“类型”推断对“中国”关键词过敏
- 位置：`parsers/guangzhou.py:374`（函数 `_map_type_from_text`）, `parsers/guangzhou.py:772`
- 规则：
  - 文本包含“中国”即偏向判定为 `央企`。
- 问题：
  - 民企/地方企业名称同样可能包含“中国”，误判概率高。
- 影响字段：`类型`
- 风险级别：中
- 建议：
  - 将“中国”从强规则改为弱信号，需与“国务院/中央/国资委”等监管词共同出现再判央企。

### 风险 F：广州从全页文本推断“隶属集团”可能吸入页面公共文案
- 位置：`parsers/guangzhou.py:44`, `parsers/guangzhou.py:402`, `parsers/guangzhou.py:756`
- 规则：
  - 无 `隶属集团` 时，从卖方文本或整页文本用“XX集团有限公司”模式提取。
- 问题：
  - 页脚、平台介绍、客服文案中的集团名可能被误抓。
- 影响字段：`隶属集团`
- 风险级别：中
- 建议：
  - 限定在结构化字段附近提取；或要求与“国家出资企业/主管部门”标签同表块出现。

### 风险 G：重庆缺编号时使用 URL `id` 伪造项目编号
- 位置：`parsers/chongqing.py:298`, `parsers/chongqing.py:300`
- 规则：
  - 无编号时从 URL 抓 `id`，写成 `CQIDxxxxx`。
- 问题：
  - `id` 是页面技术标识，不一定是业务项目编号。
- 影响字段：`项目编号`
- 风险级别：中
- 建议：
  - 将 `CQID` 放入 `备注` 或内部字段，不直接覆盖标准 `项目编号` 输出列。

### 风险 H：山东项目名称被 `<title>` 无条件覆盖
- 位置：`parsers/shandong.py:170`, `parsers/shandong.py:172`
- 规则：
  - 直接 `self.data['项目名称'] = title.get_text(strip=True)`
- 问题：
  - 站点标题/导航标题可能覆盖真实项目名。
- 影响字段：`项目名称`
- 风险级别：中
- 建议：
  - 仅在项目名为空时使用 `title`；且过滤交易所站点通用标题。

## 5. 低优先级风险

### 风险 I：交易所识别的域名兜底可受页面外链噪声影响
- 位置：`parsers/utils.py:100-112`
- 规则：
  - 基于 `snippet_lower` 中域名字符串进行最终兜底判定。
- 问题：
  - 若页面包含其他交易所脚本/引用链接，可能引起误识别（当前样本未复现）。
- 影响字段：整页 parser 路由
- 风险级别：低
- 建议：
  - 域名兜底前增加“主域来源证据”优先级（saved-from/url/meta canonical）。

## 6. 建议修复顺序
1. A（北京地区正则）与 B（北京转让方回退）  
2. C（上海融资方标签范围）  
3. D（深圳项目编号兜底收窄）  
4. E/F（广州类型与集团推断）  
5. G/H/I（其余规则收敛）

## 7. 建议验证方式（修复后）
- 构建“金样本”回归集（每交易所至少 30 页，覆盖多卖方/增资/预披露/实物资产）。
- 对以下字段做逐列比对：`项目编号`、`项目名称`、`转让方`、`隶属集团`、`类型`、`挂牌日期`。
- 针对本报告每条风险增加最小复现用例（可用简化 HTML 片段单测）。
