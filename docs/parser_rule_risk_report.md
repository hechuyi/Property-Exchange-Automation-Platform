# 解析规则风险报告

## 风险边界

解析层仍然是桌面产品最敏感的变化面，因为页面结构波动、站点兼容差异和后处理规则更新都会直接影响记录可导出状态。

## 当前主要风险

- 交易所页面结构变更导致 parser 产出字段缺失。
- OTC 或特殊页面的身份字段不足，导致记录只能进入 `failed` 或 `pending_mapping`。
- 后处理默认规则包变更后，既有映射与新记录之间出现差异。
- 解析成功但导出契约不匹配，最终体现在 `record_state` 与 `gap_codes` 上。

## 控制措施

- 通过 `tests/` 中的 parser、ingest、service、backend 契约测试锁定主链行为。
- 所有默认规则包都必须进入 repo，并通过 shipped profile 显式引用。
- 页面兼容与失败语义要优先写入公共状态契约，而不是零散地在页面逻辑中打补丁。
